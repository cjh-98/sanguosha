## 修复三国杀退出/日志/UI残留Bug（第三轮：彻底根治）

### 第一轮（已废弃）
尝试通过 _timers 追踪系统 + destroy() 方法解决，但做得不彻底。

### 第二轮
发现 3 处 game.js 核心循环中直接用 setTimeout 绕过 _timers，全部改为 this._setTimer。

### 第三轮：真正的根因
**浏览器缓存旧版 game.js**，导致 engine.destroy() 因访问不存在的新字段（_timers）而抛错，后续清理代码全部跳过，游戏继续运行。

### 修复（4处）
1. `quitToMenu()` — try-catch 包裹 engine.destroy()，加 clearTimeout(1~9999) 暴力清理
2. `restartGame()` — 同上
3. `game.js notifyState()` — 新增 if(this._destroyed) return 守卫
4. `_sgsCollectGameState()` — 改为 try-catch 包裹 engine 访问

### 核心教训
给已有项目加新 API 时，调用方必须 try-catch 兜底，防止缓存导致类型错误中断整个流程。
