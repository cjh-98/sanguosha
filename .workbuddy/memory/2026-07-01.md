## 修复"都点不动"的严重bug

### 问题现象
用户报告"都点不动"，包括"开始游戏"按钮和"测试按钮"都无法点击。

### 根本原因
在之前的编辑中，意外删除了 `openGuicai()` 和 `openGuidao()` 函数中的 `ov.innerHTML = \`` ，导致HTML代码被当作JavaScript代码执行，造成语法错误：
```
SyntaxError: Unexpected token '<'
```

这个语法错误导致整个inline script无法执行，所有全局函数（如 `goSetup()`）都没有被定义，所以所有按钮都点不动。

### 修复内容
1. **`openGuicai()` 函数**（第2076行）：添加了 `ov.innerHTML=\`` 
2. **`openGuidao()` 函数**（第2126行）：添加了 `ov.innerHTML=\``
3. **清理测试代码**：移除了"测试按钮"和 `initStatus` div

- Inline script语法检查通过
- 游戏已重新部署，等待用户测试

## 添加实时Bug日志提交系统

### 功能
1. **游戏界面添加"🐛 提交Bug"按钮**
   - 位于底部操作栏，点击弹出Bug报告窗口

2. **Bug报告窗口**
   - Bug描述输入框（可选）
   - 显示当前游戏状态摘要
   - 点击"提交日志"上传到GitHub

3. **全局函数 sgsSubmitBugLog()**
   - 游戏卡死时按 F12 打开控制台
   - 输入 `sgsSubmitBugLog()` 即可提交
   - 自动从 localStorage 读取对局日志

4. **日志内容**
   - 用户ID、对局ID
   - 当前回合、阶段、玩家
   - 牌堆/弃牌堆数量
   - 所有玩家状态（体力、手牌数等）
   - 最近100条操作日志

5. **Token处理**
   - 使用硬编码token（类似texas-poker）
   - 无需用户每次输入token

### 提交的Issue标签
- `bug-report` - Bug报告
- `auto-submit` - 自动提交
- `debug-log` - 调试日志（控制台提交时）
